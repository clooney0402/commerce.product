Why Change? (The Bottleneck)

  • 순차적 진행 (Sequential): "완벽한 기획"을 기다리느라 디자인/개발 착수가 지연됨.
  • 뒤늦은 리스크 발견: 기획이 끝나고 개발 단계에 진입해서야 기술적 이슈 확인.
  • 단일 QA의 부담: 릴리즈 직전에 모든 QA가 몰려 병목 발생.

"완벽한 기획 문서를 만드는 것보다,
빠르게 시각화하여 함께 검증하는 것이 핵심입니다."

Step 1: Fast Draft

AI-Powered Prototyping

AI 에이전트를 활용해 기획의도와 디자인 초안을 즉시 생성합니다.

🧠 ChatGPT

→ PRD 초안 생성

🎨 Gemini & Claude

Gemini Canvas로 레이아웃 시각화
→ Prototype 생성

Step 2: Pre-check

Early Sync & Validation

완성된 기획서가 아닌, '프로토타입'을 놓고 먼저 이야기합니다.

  • Design Check: Claude/Canvas가 만든 구조를 Figma로 가져와(MCP) 컴포넌트 방향성 검토.
  • Dev Check: 시각화된 화면을 보고 DB 구조, API 스펙, 기술적 리스크 사전 식별.

Goal: 개발 공수 산정의 정확도 향상 및 재작업 최소화

Step 3: Distributed QA

Layered Quality Assurance

QA를 마지막 단계가 아닌, 프로세스 전반에 분산 배치합니다.

1. 기획 QA

기획 완료 단계
자체 논리/정책 검증

2. 개발 QA

기능 개발 직후
기획 의도 싱크업

3. 릴리즈 QA

통합 테스트 &
Regression Test

Case Study & Automation

Project: 1:1 게시판 개선

MCP 연동을 통한 Fast Execution 실제 적용 사례

1. Fast Draft & Confluence MCP

Gemini Canvas에서 잡은 로직을 Confluence MCP와 연동하여,
PRD(목적, 시나리오, 정책)를 회사 템플릿에 맞춰 자동 생성 및 발행.

2. Pre-check & Figma MCP

생성된 Canvas 이미지를 Figma MCP로 전송하여,
디자인 시스템(오토레이아웃, 컴포넌트)이 적용된 편집 가능한 노드로 즉시 변환.

3. Result

기획-디자인-문서의 싱크(Sync) 비용 제로화 및 개발 착수 리드타임 50% 단축.

Productivity Hack

Hybrid Strategy: Brain & Hands

Token Limit을 극복하고 효율을 극대화하는 '생각과 실행의 분리'

🧠 Brain (두뇌)

Gemini / Claude Web (Canvas)

  • 무제한 혹은 높은 쿼터 활용
  • 고수준 사고, 로직 설계, 코드 생성 담당
"Figma용 JSON 데이터를 짜줘. 상단엔 배너, 중간엔 그리드가 들어가야 해."
→ Copy & Paste →
🙌 Hands (손)

Antigravity Agent (VS Code)

  • API 연동 및 도구 실행 권한 활용
  • 생각하지 않고 즉시 실행 (Execution Only)
"이 코드를 기반으로 지금 즉시 Figma에 적용해. 생각하지 말고 실행만 해."
💡 Tip: Agent가 기획(Brain)과 실행(Hands)을 동시에 하면 토큰이 급격히 소모됩니다. 역할을 분리하세요!
